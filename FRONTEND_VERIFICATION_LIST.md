# 前端功能验证清单 (Frontend Verification Checklist)

本文档列出了校园活动报名与签到系统中所有前端页面及核心交互功能的验证点，用于系统验收测试。

## 1. 用户认证模块 (Authentication)

### 1.1 登录页 (`/login`)

- [x] **界面展示**: 包含用户名、密码输入框及“登录”按钮。
- [x] **登录成功**: 输入正确账号 (如 `admin/admin123`)，跳转至首页。
- [x] **登录失败**: 输入错误账号，提示错误信息 (Mock 或 真实接口返回)。
- [x] **跳转注册**: 点击“注册新账号”链接，正确跳转至注册页。

### 1.2 注册页 (`/register`)

- [x] **表单填写**: 输入用户名、真实姓名、密码。
- [x] **注册逻辑**: 点击注册后，若成功应弹窗提示并跳转回登录页。
- [x] **错误处理**: 输入重复用户名，应提示“用户名已存在”。

### 1.3 退出登录

- [x] **位置**: 首页顶部导航栏。
- [x] **逻辑**: 点击“退出”后，清除会话引用，并强制跳转回登录页。

---

## 2. 普通用户功能 (User Features)

### 2.1 首页/活动列表 (`/`)

- [x] **欢迎语**: 顶部显示当前登录用户的真实姓名。
- [x] **列表加载**: 能正常加载活动卡片列表。
- [x] **卡片信息**: 每个卡片显示标题、时间、地点、描述（超出截断）。
- [x] **查看详情**: 点击“查看详情”按钮，正确跳转至 `/events/{id}`。

### 2.2 活动详情页 (`/events/{id}`)

- [x] **信息展示**:
  - [x] 标题、地点、活动时间、签到窗口时间显示无误。
  - [x] 状态徽章（进行中/即将开始/已结束）根据当前时间正确显示。
- [x] **报名流程**:
  - [x] **未报名状态**: 显示红色大号“立即报名”按钮。
  - [x] **点击报名**: 提示成功，按钮区域变为“✅ 您已报名”。
- [x] **取消报名**:
  - [x] **已报名且未签到**: 显示红色边框“取消报名”按钮。
  - [x] **点击取消**: 弹出二次确认框，确认后状态恢复为“立即报名”。
- [x] **签到流程 (核心)**:
  - [x] **状态检查**: 仅在“已报名”状态下可见签到区域。
  - [x] **时间窗外**: 签到按钮禁用 (Disabled)，文案显示“签到未开始”或“签到已截止”。
  - [x] **时间窗内**: 签到按钮可用 (亮黄色)，文案显示“立即签到”。
  - [x] **点击签到**: 点击后提示成功，按钮区域变为“🎉 已签到”，且**“取消报名”按钮消失** (已签到不可取消)。

---

## 3. 管理员功能 (Admin Features)

### 3.1 导航与入口

- [x] **权限控制**: 已实现 Admin/User 布局分离。管理员登录后自动跳转 `/admin`，普通用户跳转 `/user`。若普通用户尝试访问 `/admin` 路由，会被强制重定向回用户首页。
- [x] **进入后台**: 管理员登录后直接进入后台界面。普通用户无法通过 UI 或 URL 进入后台。

### 3.2 活动管理列表 (`/admin`)

- [x] **列表展示**: 表格显示 ID、标题、活动时间段、签到窗口、地点。
- [x] **发布活动**:
  - [x] 点击“+ 发布新活动”，弹出 Modal。
  - [x] 填写表单（含签到开始/结束时间）并保存。
  - [x] 列表自动刷新，显示新活动。
- [x] **编辑活动**:
  - [x] 点击“编辑”，Modal 回显活动信息。
  - [x] 修改并保存，列表信息更新。
- [x] **删除活动**:
  - [x] 点击“删除”，弹出二次确认。
  - [x] 确认后活动从列表中移除。

### 3.3 查看名单 (`Roster Modal`)

- [x] **入口**: 点击活动行上的蓝色“名单”按钮。
- [x] **数据展示**:
  - [x] 显示所有相关用户的姓名。
  - [x] **报名时间**: 格式化显示。
  - [x] **签到状态**:
    - [x] 未签到用户显示黄色“未签到”，签到时间为 `-`。
    - [x] 已签到用户显示绿色“已签到”，显示具体签到时间。
- [x] **只读确认**: 确保该弹窗内无任何修改操作按钮。

---

## 4. 异常与边界测试 (Edge Cases)

- [ ] **服务器重连**: 把后端重启，前端操作应在失败后（如下次点击时）报错。
- [ ] **无数据**: 数据库为空时，列表应显示为空白或“暂无数据”。
- [ ] **401 处理**: Cookie 过期后，点击任何请求按钮应自动重定向至登录页 (依赖 Axios 拦截器)。
